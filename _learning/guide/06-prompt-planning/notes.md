## 1. 프롬프트 기획의 필요성

### 기획해야 하는 세 가지 이유

## 2. 사용자 세그먼테이션

### 대화 분석 4축

| 축 | 구분 | 설명 |
|---|------|------|
| **Turn** | 싱글턴 vs 멀티턴 | 대화 횟수 |
| **Action** | 정보 검색 vs 다른 행위 | 사용 목적 |
| **Structure** | 선호 vs 비선호 구조 | 대화 구조 |
| **Stance** | 감정적 vs 비감정적 | 감정 표현 |

### 세그먼트 분류

```
SI   = Single-Turn, Information Seeking (정보 검색형)
SO   = Single-Turn, Other

MISP = Multi-Turn, Information, Stance Preferred (만족)
MISD = Multi-Turn, Information, Stance Dispreferred (불만족)
MISU = Multi-Turn, Information, Stance Unknown (알수없음)

MOSP = Multi-Turn, Other, Stance Preferred
MOSD = Multi-Turn, Other, Stance Dispreferred
MOSU = Multi-Turn, Other, Stance Unknown
```

### 세그먼트별 특징

| 세그먼트 | 특징 | 프롬프트 전략 |
|---------|------|--------------|
| **SI** | 정보 검색형, 즉흥적 | 검색 엔진 연동, 멀티턴 유도 |
| **MISP/MOSP** | AI 친화적, 긍정적 | Gamification, 페르소나 |
| **MISU/MOSU** | 아이디어 탐구형 | 가능성 탐구 지원 |

### 세그먼트 활용법

1. 사용자 발화 데이터 정량화 (LLM 분류)
2. Heavy User 분석
3. 세그먼트별 A/B Test
4. Low Retention 그룹 원인 분석
5. 초개인화 맞춤형 AI 경험 설계

## 3. 기획 접근 방법론

### 연역적 접근

```
가설 설정 → 관찰/실험 → 검증
```

| 장점 | 단점 |
|------|------|
| 논리적, 체계적 | 실제와 괴리 가능 |
| 시간 효율적 | 제한된 유연성 |
| 명확한 가정 기반 | 피드백 반영 어려움 |

**활용**:
- 이론적 모델 수립
- 규칙 기반 시스템 구축
- 가설 기반 기능 설계

### 귀납적 접근

```
관찰/실험 → 결론/일반화
```

| 장점 | 단점 |
|------|------|
| 실제 데이터 반영 | 시간 소요 |
| 사용자 행동 반영 | 데이터 편향 가능 |
| 피드백 기반 개선 | 새 트렌드 반영 한계 |

**활용**:
- 사용자 데이터 분석
- 피드백 반영
- A/B 테스트

### 실전 예시: SI/SO 유형이 많은 이유

**연역적 접근**:
- 가설1: 사용자가 생성형 AI를 어려워한다
- 가설2: 프롬프트 작성을 귀찮아한다
- 가설3: 정보 검색이 주요 동기다

**귀납적 접근**:
- 관찰1: 정보 검색 유형이 가장 많음
- 관찰2: 프롬프트 길이가 짧고 불충분함
- 관찰3: 한 번 대화 후 이탈

## 4. 핵심 이론

### Nudge Theory (행동 경제학)

> Thaler & Sunstein, 2008

**핵심 원리**:
- **선택의 자유**: 개인 선택권 제한 없음
- **작은 개입**: 사소한 변화로 큰 영향
- **예측 가능한 비합리성**: 인간 행동 패턴 활용
- **선택 설계**: 환경 조정으로 더 나은 선택 유도

**프롬프트 적용**:
- 후속 질문에 넛지 적용
- 멀티턴 유도 장치
- 사용자 만족 극대화

### Theory of Mind (마음 이론)

> Premack & Woodruff, 1978

**정의**: 다른 사람의 정신 상태를 이해하고 추론하는 능력

**핵심**:
- 타인이 다른 생각/믿음/욕구를 가질 수 있음 인식
- 타인의 행동 예측 및 설명

**프롬프트 적용**:
- 사용자 의도 추론
- 맞춤형 응답 생성
- 대화 흐름 예측

## 5. 프롬프트 기획 실습 3종

### 5.1 프롬프트 질문 생성기

**문제**: 싱글턴 사용자 비율이 높음, 서비스 이용 시간 짧음

**해결 전략**:
- Static → Dynamic Question Generator
- FAQs: Prediction Probability
- Nudge Theory 적용

**구현 아이디어**:
- 사용자 질문 기반 후속 질문 제공
- UI 개선으로 멀티턴 유도

### 5.2 프롬프트 자동 완성기

**문제**: 사용자 프롬프트가 구체적이지 않음, 보조 장치 부재

**해결 전략**:
- Intent → User Prompt 변환
- 효율성 극대화

**기대 효과**:
- 사용자 만족도 증가
- 리텐션 향상
- 더 복잡하고 정교한 질문으로 발전

### 5.3 시스템 프롬프트 개선

**문제**: 특정 세그먼트 비율 편중, 시스템 프롬프트 문제

**글로벌 이슈**:
- 프롬프트가 너무 일반적
- 회사 정체성 미반영
- 특징 있는 답변 부재

**로컬 이슈**:
- 실시간 정보 미제공
- 텍스트 가독성 저하
- 번역체 한국어
- 부정확/거짓 정보 지침 부재

**해결 방법**:
- 파레토 법칙 (Heavy User 20% 분석)
- 사용자 경험 최적화
- 다양한 요구 수용

